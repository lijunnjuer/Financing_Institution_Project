## 全局制作规范

- **配色**：深蓝(#1A3A5C)为主色，橙色(#ED7D31)为强调色，灰色(#A5A5A5)为辅助色
- **字体**：标题用微软雅黑加粗，正文用微软雅黑常规，数据用Arial或等宽字体
- **背景**：白色底或浅灰(#F5F5F5)底，避免深色背景（教室投影仪对比度不足）
- **每页**：右下角统一放页码，左上角统一放课题简称"算法锁定与专家判断"
- **截图规范**：所有PDF截图需放大到清晰可读，关键数字用红框标出
- **动画**：尽量不用动画（教室电脑性能不确定），用静态排版替代

- 截图放大到清晰可读

**右侧文字**（占60%宽度）：
```
Stein (2002) Journal of Finance

核心命题：
  软信息（不可验证·传递失真）
    → 需要分权 → 赋权一线人员决策

  硬信息（可验证·标准化）
    → 适合集权 → 层级组织有效传递

实证证据：
  大银行收购小银行后
  → 中小企业贷款显著下降
  （小企业贷款依赖软信息，
    大银行层级不适合处理软信息）

应用于本课题：
  沪科积分 = 硬信息工具（信用评分技术）
  科技企业 = 最需要软信息的对象（关系型贷款）
  → 结构性矛盾
```

**制作**：
- 左侧截图顶部标注"Stein(2002)原文摘要"
- 右侧文字用箭头串联推导逻辑
- 最后两行"结构性矛盾"用橙色加粗

**耗时**：1.5分钟

### 第8页：纯量化模型的结构性天花板（关键证据一）

**布局**：上半部分放BIS论文关键信息，下半部分放柱状对比图

**上半文字**：
```
BIS Working Paper No.834
Gambacorta, Huang, Qiu & Wang (2020)

研究设计：
  样本：中国某金融科技公司 180万笔贷款
  方法：传统银行模型 vs 金融科技模型 对比

核心发现：
  传统银行模型（仅财务数据+征信报告）
    → AUC = 56.45% ~ 65.36%

  金融科技模型（+实时交易流水+网络行为数据）
    → AUC = 88.81%

  23个百分点的提升 = 来自额外数据类型
                      ≠ 来自算法改进
```

**下半部分**：
- 制作一个柱状对比图（直接在PPT里用形状画，不用导入外部图表）
- 左柱：标注"传统硬信息模型"，高度约65%，颜色灰色
- 右柱：标注"加入行为数据后"，高度约89%，颜色深蓝
- 两柱之间用橙色箭头标注"+23pp"
- Y轴标注"AUC（预测准确率）"，范围0-100%

**截图**：
- 打开 `参考文献/学术论文/BIS_2020_ML_NonTraditional_Data_Credit_Scoring_No834.pdf`
- 截取包含AUC对比数据的段落
- 放在柱状图下方作为来源标注

**底部结论**（橙色加粗）：
```
输入仅限于传统硬信息时，AUC上限约65%，与算法无关。
不是模型不够好，是数据类型不够多。
```

**耗时**：1.5分钟（这是最核心的证据页，要讲清楚）

### 第9页：国内实证印证（关键证据二）

**布局**：左侧放论文截图，右侧放对比表

**左侧**（占40%宽度）：
- 打开 `参考文献/学术论文/科技型中小企业融资信用风险评估研究——基于GA-PSO算法的建模与优化_夏轶群.pdf`
- 翻到第3页，截取表1（三种模型准确率对比表）
- 用红框标出"多元回归模型 60%"和"GA-PSO 80%"

**右侧文字**：
```
夏轶群、王圣权 (2016)
《会计之友》第16期

研究设计：
  64家深市中小板科技企业（2014年末数据）
  三种模型对比：多元回归 vs GP算法 vs GA-PSO

关键发现：

  多元回归模型    测试准确率 60%
  GP算法         测试准确率 70%
  GA-PSO混合算法  测试准确率 80%

  → 即使是优化后的算法，用传统硬信息
    预测科技企业信用风险，准确率天花板仅80%
  → 科技企业的硬信息质量比传统企业更差
```

**底部结论**（橙色加粗）：
```
科技企业的硬信息对创新潜力的反映能力更弱，
纯量化模型的预测力更低。
```
